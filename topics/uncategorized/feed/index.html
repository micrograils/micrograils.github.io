入门准备

1. 环境准备

JDK（8+）

https://www.oracle.com/technetwork/java/javase/downloads/index.html

Grails（4.*）

http://docs.micrograils.com/latest/guide/gettingStarted.html#requirements

Groovy（本次非必须，长期很有用）

http://www.groovy-lang.org/install.html

2. 验证环境

[Aaron.Xu@micrograils.com]~$ java -versionjava version "1.8.0_201"Java(TM) SE Runtime Environment (build 1.8.0_201-b09)Java HotSpot(TM) 64-Bit Server VM (build 25.201-b09, mixed mode)[Aaron.Xu@micrograils.com]~$ grails -v| Grails Version: 4.0.0| JVM Version: 1.8.0_201[Aaron.Xu@micrograils.com]~$ groovy -vGroovy Version: 2.5.6 JVM: 1.8.0_201 Vendor: Oracle Corporation OS: Mac OS X

Hello World

1. 创建项目

打开命令行，输入命令（注1）：

grails create-app helloworld

示例：

[Aaron.Xu@micrograils.com]~/workspaces$ grails create-app helloworld| Application created at /Users/xuh/workspaces/helloworld

helloworld项目创建完毕。

2. 编写代码

继续在当前的命令行状态下，输入（注2）：

cd helloworldgrails

第一允许需要一点点小耐心，等待各种依赖自动下载完毕，直到出现 grails> 提示符：

[Aaron.Xu@micrograils.com]~/workspaces$ cd helloworld/| Resolving Dependencies. Please wait...| Starting interactive mode...| Enter a command name to run. Use TAB for completion:grails>

输入命令 create-controller helloworld 来创建我们的第一个 controller（注3）：

grails> create-controller helloworld| Created grails-app/controllers/helloworld/HelloworldController.groovy| Created src/test/groovy/helloworld/HelloworldControllerSpec.groovygrails>

用你习惯的文本编辑器编辑打开 HelloworldController.groovy，例如（注4）：

grails> !open -a atom grails-app/controllers/helloworld/HelloworldController.groovy

加入 render "hello, world."，修改结果如下：

package helloworldclass HelloworldController {    def index() {        render "hello, world."    }}

查看效果

1. 运行项目

继续在当前命令行，执行 run-app 命令：

grails> run-app| Running application...Grails application running at http://localhost:8080 in environment: development<==========---> 83% EXECUTING [26s]> :bootRungrails>

2. 查看效果

新Term窗口，命令行：

curl http://127.0.0.1:8080/helloworldhello, world.

或打开浏览器， 访问 http://127.0.0.1:8080 查看页面效果。

相关讲解

注1:

grails 创建项目的命令，Grails 4 目前支持多种 profile，可以在创建时指定要创建的项目都类型，具体信息，参考：

http://docs.micrograils.com/latest/guide/profiles.html

注2:

Grails 的命令行操作很强大，配上一个趁手的文本编辑器，完全可以实现整个项目的命令行开发。关于命令行的详细信息，请参考：

http://docs.micrograils.com/latest/guide/commandLine.html

注3:

类似于 create-* 的命令还有很多，可以在 Interactive 模式下输入 create- 后按 tab 键查看更多命令。更多关于此命令的介绍，请参考：

http://docs.micrograils.com/latest/ref/Command%20Line/create-controller.html

注4:

这个是 Grails 4 里，很帅气的一个「本地命令调用」支持方式。感叹号 ！意思是调用本地命令，尤其在临时需要执行一个本地命令时、完全不用新开一个窗口或推出 Grails 开发模式，本例中的 open -a *** 则是 Mac 的命令行用法，作用是从命令行直接打开一个应用。

本行代码的意思为：调用本地命令 open 使用 atom 编辑器打开指定的文件。

结束语

通过基础环境和 helloword 项目的创建和运行，应该能够清晰感受到 Grails 非常高效的开发效率，后续的系列文章中，讲进一步介绍 Grails / Groovy 一些更加精妙的设计和实现。敬请期待。

]]> 73 Grails 4 GA 发布 https://www.micrograils.com/post/20190723144305.html Tue, 23 Jul 2019 06:43:05 +0000 https://www.micrograils.com/?p=47 来自 MicroGrails.com – Grails微服务社区

作者：Graeme Rocher，OCI Grails＆Micronaut产品负责人和首席软件工程师。

在几位发布候选人和来自社区的积极反馈之后，来自Object Computing，Inc（OCI）的 Grails 团队很高兴地宣布了 Grails 4 的GA版。

Grails 4 代表了一次重大的升级，现在框架在更新的最小依赖项集之上进行了重构，包括：

  • Spring 5.1
  • Spring Boot 2.1
  • Java 8
  • Hibernate 5.4
  • Groovy 2.5
  • Gradle 5

此外，Micronaut 现在作为 Grails 的一部分作为父上下文包含在内，允许在Grails 4 应用程序中使用任何 Micronaut 功能，无论是声明性HTTP客户端还是Kafka客户端。事实上，Micronaut HTTP客户端 现在被认为是在Grails应用程序中使用的受支持的HTTP客户端。

由于 Spring Boot 2.1 的变化以及 Grails 的一些内部基础实现迁移到 Micronaut，启动时间和内存消耗也比以前的版本有了显着优化和改进。

许多 Grails 插件已经过验证可以与 Grails 4 一起使用。对于 Grails 插件社区内的人来说，现在是时候升级你的插件并验证他们是否继续使用 Grails 4 ，因为这是一个新的主版本，所以已经提供最新版 API升级指南，以及包含 Spring Boot，Groovy 和 Hibernate 在内的环境依赖可能需要改变的内容。具体操作请阅读升级文档以获取更多信息。

感谢所有为此次发布做出贡献的人，我们会有更多有关 Grails  和  Micronaut  计划的精彩公告  。
